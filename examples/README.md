# 示例项目

这里是使用 Product Context 的示例项目，展示如何在实际项目中填写产品上下文。

## 示例

### vibling/
**项目类型**: 音乐可视化 Web 应用

展示了：
- 清晰的目标用户定义
- 明确的非目标（避免scope creep）
- 阶段性路线图
- 品牌调性指南
- 实际的MEMO状态

### fictional-saas/
**项目类型**: 发票催收 SaaS 工具

展示了：
- SaaS商业模式设计
- 催款邮件的品牌调性（敏感场景）
- 技术决策记录
- 待验证假设

## 如何使用这些示例

1. **学习**: 阅读示例，了解如何填写各字段
2. **参考**: 复制相似项目类型的示例作为起点
3. **适配**: 根据你的项目调整内容

## 真实项目 vs 示例

示例项目的内容是模拟的，但结构和写法是真实可用的。

在真实项目中：
- 信息会更详细
- MEMO会频繁更新
- 决策记录会随项目演进
