# InvoiceFlow 路线图

## 当前阶段
MVP - 验证核心价值

## 里程碑

### Phase 1: MVP (Week 1-4)
**目标**: 手动添加发票，自动提醒

- [x] 用户注册/登录
- [x] 手动添加发票
- [ ] 发票状态管理
- [ ] 到期日提醒（邮件）
- [ ] 简单Dashboard

**技术栈**:
- Next.js + Prisma + PostgreSQL
- Resend（邮件发送）
- Vercel部署

**完成标准**: 5个用户愿意每天使用

### Phase 2: 自动化 (Week 5-8)
**目标**: 自动催款

- [ ] 催款邮件模板
- [ ] 自动发送逻辑（过期后1天、7天、14天）
- [ ] 发票工具导入（Stripe, PayPal）
- [ ] 邮件打开/点击追踪

### Phase 3: 商业化 (Week 9-12)
**目标**: 验证付费意愿

- [ ] 付费套餐设计
- [ ] Stripe订阅集成
- [ ] 套餐限制逻辑
- [ ] 账单管理

### Phase 4: 增长 (Month 4+)
**目标**: 规模化

- [ ] 更多发票工具集成
- [ ] AI优化催款文案
- [ ] 客户付款概率预测
- [ ] 推荐奖励系统

### 未来考虑
- 团队版
- API开放
- 移动App
- 多语言支持

## 关键假设待验证
1. 自由职业者愿意用工具管理发票
2. 自动催款不会伤害客户关系
3. $9/月是可接受的价格点
