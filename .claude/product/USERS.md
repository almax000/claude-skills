# Users

## Primary Persona: Claude Code Newcomer

**Profile:**
- Just started using Claude Code
- Wants to customize but doesn't know where to start
- Overwhelmed by the options: agents, hooks, rules, settings

**Pain Points:**
- "What's the difference between agents and hooks?"
- "How do I create a skill that actually works?"
- "Where do I put these files?"

**What They Need:**
- Ready-to-use skills they can copy and run
- Skills that generate other configurations
- Clear examples to learn from

## Secondary Persona: Individual Developer

**Profile:**
- Using Claude Code daily for personal projects
- Wants to be more productive
- Has specific workflows they repeat often

**Pain Points:**
- Repetitive tasks (commit, review, debug)
- Inconsistent practices across projects
- Time spent configuring each new project

**What They Need:**
- Skills for common workflows
- Project initialization skills
- Customization patterns to adapt

## Tertiary Persona: Small Team Lead

**Profile:**
- Leading a small team (2-5 developers)
- Wants consistent practices across the team
- Needs to onboard new team members quickly

**Pain Points:**
- Different team members using different approaches
- New members don't know team conventions
- Hard to share and update configurations

**What They Need:**
- Shareable skill collections
- Skills that enforce team standards
- Easy installation and updates
