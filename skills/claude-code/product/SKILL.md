# /product

分析当前对话，智能更新 `.claude/product/` 文件。

## 执行流程

### 1. 检查是否已初始化

```
.claude/product/ 存在？
├─ 否 → 提示："请先运行 /product-init 初始化产品文档"，结束
└─ 是 → 继续
```

### 2. 读取所有产品文档

读取 `.claude/product/` 目录下的所有文件。

### 3. 分析对话历史

识别产品层面的变化：

| 对话内容 | 更新文件 |
|---------|---------|
| 阶段完成、里程碑达成、重要决策 | ROADMAP.md |
| 产品方向/定位变化 | VISION.md |
| 用户理解变化 | USERS.md |
| 功能边界变化（新增功能/不做什么） | SCOPE.md |
| 其他话题 | 对应扩展文件 |

### 4. 展示变更

- 列出需要更新的文件
- 展示具体的 diff（旧 → 新）
- 如果讨论了某话题但对应文件不存在，提议创建

### 5. 用户确认后执行

- 确认 → 执行更新
- 拒绝 → 取消
- 部分确认 → 只更新选中的

### 6. 结束提示

```
已更新 .claude/product/ROADMAP.md, .claude/product/SCOPE.md

下次完成里程碑或做出重要决策后，可以再次运行 /product
```

## 智能判断原则

- 只识别**产品/商业层面**的变化
- 技术实现细节不触发更新
- 没有变化时，告知用户"本次对话没有需要更新的产品决策"

## 文件职责

| 文件 | 核心问题 |
|------|----------|
| **ROADMAP.md** | 当前阶段？下一步？为什么这样决定？ |
| VISION.md | 为什么做？使命、愿景、价值主张 |
| USERS.md | 给谁做？用户画像、痛点、场景 |
| SCOPE.md | 做什么？功能边界、明确不做什么 |

**扩展文件**（按需添加）：BRAND.md, MARKETING.md, BUSINESS.md, METRICS.md...

## 注意事项

1. **展示 diff** — 修改前必须展示变更，用户确认后执行
2. **决策 vs 操作** — 只记录产品决策，技术细节放 CLAUDE.md
3. **版本控制** — 这些文件应该和代码一起提交

## ROADMAP 实时更新

除了用户调用 `/product`，AI 也可以**主动提议**更新 ROADMAP.md。

**触发条件**（AI 主动识别）：
- 完成了一个功能/阶段
- 做出了重要决策
- 改变了计划方向
- 遇到了阻碍/调整

**行为**：
```
AI: "我注意到你刚才决定了 X，要不要更新 ROADMAP？"
用户: 好 / 不用
```

**不需要用户调用 /product**，AI 在对话中主动提议。但仍需用户确认后才执行更新。
