# Product Context

This project uses `.product/` directory for product/business decisions.

## Why This Project Exists

See `.product/VISION.md` for:
- Mission and vision
- Value proposition
- Target market

## Target Users

See `.product/USERS.md` for:
- User personas
- Pain points
- Usage scenarios

## Feature Scope

See `.product/SCOPE.md` for:
- Core features (must have)
- Non-goals (explicitly not doing)
- Constraints

## Current Phase

See `.product/ROADMAP.md` for:
- Current development phase
- Milestones
- Decision log

## Extensions (if exist)

- `.product/BRAND.md` - Voice & tone guidelines
- `.product/MARKETING.md` - User acquisition strategies
- `.product/BUSINESS.md` - Pricing & monetization
- `.product/OPS.md` - Infrastructure decisions

## Guidelines

1. Before implementing features, read `VISION.md` + `SCOPE.md`
2. When writing user-facing text, follow `BRAND.md` (if exists)
3. Check `ROADMAP.md` for current priorities
4. Features in SCOPE.md "Non-Goals" should be flagged
