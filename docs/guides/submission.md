# Submitting to Skill Aggregation Platforms

This guide explains how to submit Product Context to skill aggregation platforms once you have established community usage.

## VoltAgent/awesome-agent-skills

### Prerequisites

1. Public repository with working skill ✅
2. Documentation (README or SKILL.md) ✅
3. **Real community usage** - brand-new skills are not accepted

### Submission Steps

1. Fork [VoltAgent/awesome-agent-skills](https://github.com/VoltAgent/awesome-agent-skills)

2. Add entry to README.md under appropriate category:
   ```markdown
   - **[almax000/product-context](https://github.com/almax000/product-context/tree/main/skill)** - Product decisions for AI coding assistants
   ```

3. Create PR with title: `Add skill: almax000/product-context`

### Checklist Before Submission

- [ ] GitHub stars > 10 (demonstrates interest)
- [ ] At least 3 external users (not just yourself)
- [ ] Skill tested and working in Claude Code
- [ ] README clearly explains usage

## anthropics/skills

The official Anthropic skills repository is curated by Anthropic staff. Contact Anthropic directly if you want to be featured.

## Other Platforms

### Product Hunt

Consider launching on Product Hunt as "developer tools" category once you have:
- Working demo
- Clear value proposition
- README with screenshots/GIFs

### Hacker News

Share on HN Show when you have a compelling story about why you built this.

## Promotion Strategy

1. **Week 1-2**: Use in your own projects, refine based on experience
2. **Week 3-4**: Share on Twitter/X, Reddit (r/ClaudeAI, r/ArtificialIntelligence)
3. **Month 2**: Write a blog post explaining the methodology
4. **Month 3+**: Submit to aggregation platforms with usage evidence
