# VISION.md 格式规范

**版本**: v1
**状态**: 稳定
**类型**: 核心文件

## 概述

VISION.md 定义产品的使命、愿景和价值主张，回答"为什么做这个产品"。

这是决策层文档，聚焦于商业/产品方向，不包含技术实现细节。

## 必需字段

```markdown
# 产品名称

## 使命
<!-- 我们为什么存在？解决什么问题？ -->

## 愿景
<!-- 如果成功，世界会变成什么样？ -->

## 价值主张
<!-- 用户为什么选择我们而不是竞品？ -->
```

## 推荐字段

```markdown
## 核心洞察
<!-- 我们看到了什么别人没看到的？ -->

## 差异化
<!-- 与竞品的关键区别 -->
```

## 示例

```markdown
# Vibling

## 使命
让每个音乐人都能轻松制作专业的音乐可视化视频，无需学习复杂软件。

## 愿景
成为独立音乐人发布音乐时的首选可视化工具，让"音乐+视觉"成为标配。

## 价值主张
- **vs After Effects**: 3分钟 vs 3小时，零学习成本
- **vs 其他工具**: 专为音乐人设计，不是通用视频工具
- **vs 外包**: 自己动手，随时修改，成本为零

## 核心洞察
音乐人在社交媒体发布音乐时需要视觉内容，但现有工具要么太复杂（AE）要么不够专业（免费工具）。

## 差异化
- 专注音乐可视化这一个场景
- 模板基于音乐类型设计，不是通用模板
- 支持音频响应动画
```

## AI 使用指南

当 AI 读取 VISION.md 时，应该：

1. **对齐方向** - 确保实现的功能服务于使命和愿景
2. **传达价值** - 在文案中体现价值主张
3. **保持聚焦** - 如果功能偏离使命，应该提醒用户
4. **引用洞察** - 做决策时参考核心洞察
