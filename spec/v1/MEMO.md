# MEMO.md 格式规范

**版本**: v1
**状态**: 稳定

## 概述

MEMO.md 是AI的"短期记忆"，用于记录当前工作状态，帮助在新会话中快速恢复上下文。

这是唯一一个**AI应该主动更新**的文件。

## 推荐字段

```markdown
# 当前状态

## 上次会话
<!-- 上次做了什么，简短描述 -->

## 当前任务
<!-- 正在解决什么问题 -->

## 下一步
<!-- 计划做什么 -->

## 重要决策
<!-- 最近做的关键决策和原因 -->

## 注意事项
<!-- AI需要知道的约束或警告 -->
```

## 可选字段

```markdown
## 阻塞问题
<!-- 遇到的问题，需要人工介入 -->

## 技术债务
<!-- 临时方案，需要后续优化 -->

## 参考链接
<!-- 相关的文档、Issue、讨论 -->
```

## 示例

```markdown
# Vibling 当前状态

**更新时间**: 2025-01-31

## 上次会话
完成了音频上传功能，包括文件验证和存储。

## 当前任务
实现模板系统的基础架构。

## 下一步
1. 设计模板数据结构
2. 实现模板渲染引擎
3. 添加3个默认模板

## 重要决策
- **模板格式**: 选择JSON而非YAML，因为前端解析更方便
- **渲染引擎**: 使用Canvas而非WebGL，降低实现复杂度

## 注意事项
- 视频导出功能先不做，留到Phase 2
- 目前只支持MP3格式，WAV支持延后

## 技术债务
- `audioProcessor.ts` 中的频率分析算法需要优化，目前性能一般
- 错误处理还不完善，先用console.error
```

## AI使用指南

当AI读取MEMO.md时，应该：

1. **恢复上下文** - 了解之前做了什么，避免重复工作
2. **继续任务** - 从"下一步"继续
3. **遵守决策** - 不要质疑"重要决策"中的选择，除非用户主动提出
4. **注意约束** - "注意事项"中的内容是硬约束

当AI需要更新MEMO.md时：

1. **会话结束时** - 总结本次做了什么
2. **做出重要决策时** - 记录决策和原因
3. **发现问题时** - 记录到"阻塞问题"或"技术债务"
4. **保持简洁** - 这不是完整日志，只记关键信息
