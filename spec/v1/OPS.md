# OPS.md 格式规范

**版本**: v1
**状态**: 稳定
**类型**: 扩展文件

## 概述

OPS.md 定义运营基础设施决策，回答"怎么运行"。

**重要**: 这是决策层文档，记录"选什么"和"为什么"，不是"怎么操作"。

具体的部署步骤、配置命令应该放在 AGENTS.md 或 docs/。

## 必需字段

```markdown
# 运营决策

## 基础设施
<!-- 选择什么服务，为什么 -->

## 成本结构
<!-- 预计成本 -->
```

## 推荐字段

```markdown
## 扩展触发器
<!-- 什么情况下需要升级 -->

## 监控告警
<!-- 需要监控什么 -->

## 灾难恢复
<!-- 备份和恢复策略 -->
```

## 示例

```markdown
# Vibling 运营决策

## 基础设施

| 组件 | 选择 | 原因 |
|------|------|------|
| 托管 | Vercel | 免费额度足够 MVP |
| 数据库 | Supabase | Auth + DB + Storage 一体 |
| 视频处理 | 浏览器端 FFmpeg.wasm | 省服务端成本 |
| 文件存储 | Supabase Storage | 与数据库同一服务 |
| 域名 | Cloudflare | 免费 DNS + CDN |

## 成本结构

### 当前（MVP 阶段）
| 项目 | 月费 | 备注 |
|------|------|------|
| Vercel | $0 | Free tier |
| Supabase | $0 | Free tier |
| 域名 | $12/年 | .com |
| **总计** | **~$1/月** | |

### 预计（1000 用户）
| 项目 | 月费 | 备注 |
|------|------|------|
| Vercel | $0 | 静态站点仍够用 |
| Supabase | $25 | Pro plan |
| **总计** | **~$25/月** | |

## 扩展触发器

| 指标 | 阈值 | 行动 |
|------|------|------|
| DAU | > 500 | 考虑 Supabase Pro |
| 存储 | > 1GB | 升级存储计划 |
| API 调用 | > 50k/月 | 考虑加缓存 |
| 导出视频大小 | 服务端需求 | 考虑云函数 |

## 监控告警

### 需要监控
- 服务可用性（Vercel/Supabase）
- 数据库连接数
- 存储使用量
- API 错误率

### 工具选择
- 使用 Supabase 自带监控
- 免费的 UptimeRobot 监控可用性

## 灾难恢复

### 备份策略
- 数据库：Supabase 自动备份（每日）
- 代码：GitHub 仓库
- 配置：.env.example 文档化

### 恢复时间目标
- RTO: < 4 小时（可接受手动恢复）
- RPO: < 24 小时（日备份）
```

## 决策 vs 操作 边界

### 属于 OPS.md（决策）
- 选择 Vercel 而非 AWS（以及原因）
- 使用 Supabase Free tier
- 扩展触发条件

### 不属于 OPS.md（操作）
- 如何配置 Vercel 项目
- Supabase 表结构设计
- 部署命令
- 环境变量设置

操作类内容应该放在：
- `AGENTS.md` - AI 代理需要知道的操作
- `docs/deployment.md` - 人类需要的部署文档
- `README.md` - 快速开始

## AI 使用指南

当 AI 读取 OPS.md 时，应该：

1. **成本意识** - 选择符合当前阶段的方案
2. **触发器检查** - 实现功能时考虑是否触发扩展
3. **不随意升级** - 遵守成本约束，不主动建议高成本方案
4. **记录决策** - 基础设施变更时更新此文件
