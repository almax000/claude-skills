# PRD.md 格式规范

**版本**: v1
**状态**: 稳定

## 概述

PRD.md（Product Requirements Document）是产品的灵魂文件，定义产品是什么、给谁用、解决什么问题。

## 必需字段

```markdown
# 产品名称

## 一句话描述
<!-- 用一句话说清楚这是什么，不超过100字 -->

## 目标用户
<!-- 谁会用这个产品？他们的痛点是什么？ -->

## 核心价值主张
<!-- 为什么用户要用你的产品而不是其他的？ -->

## 核心功能
<!-- 产品的主要功能列表，建议3-5个 -->
```

## 推荐字段

```markdown
## 非目标
<!-- 明确不做什么，防止scope creep -->

## 成功指标
<!-- 如何衡量产品是否成功？ -->

## 技术约束
<!-- 影响产品决策的技术限制 -->
```

## 示例

```markdown
# Vibling

## 一句话描述
让音乐可视化创作像做PPT一样简单的Web应用。

## 目标用户
- 想为自己音乐作品制作可视化视频的独立音乐人
- 没有编程或视频制作经验
- 预算有限，无法雇佣专业团队

## 核心价值主张
无需学习复杂软件，5分钟内就能为任何音乐生成专业的可视化视频。

## 核心功能
1. 音频上传与分析
2. 模板选择与自定义
3. 实时预览
4. 一键导出视频

## 非目标
- 不做专业视频编辑（不是Premiere/Final Cut竞品）
- 不做音频编辑
- 不做社交功能
```

## AI使用指南

当AI读取PRD.md时，应该：

1. **功能决策** - 在实现新功能前，检查是否在"核心功能"列表中
2. **说不** - 如果请求在"非目标"中，应该提醒用户
3. **用户视角** - 写代码时考虑"目标用户"的技术水平
4. **价值对齐** - 确保实现方式符合"核心价值主张"
