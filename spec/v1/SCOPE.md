# SCOPE.md 格式规范

**版本**: v1
**状态**: 稳定
**类型**: 扩展文件

## 概述

SCOPE.md 定义产品的功能边界，回答"做什么/不做什么"。

这是防止 scope creep 的关键文档，明确列出非目标。

## 必需字段

```markdown
# 功能范围

## 核心功能
<!-- 必须做的功能 -->

## 非目标
<!-- 明确不做的功能 -->
```

## 推荐字段

```markdown
## 可选功能
<!-- 如果时间允许可以做 -->

## 延后功能
<!-- 未来可能做，现在不做 -->

## 技术约束
<!-- 技术上的限制 -->
```

## 示例

```markdown
# Vibling 功能范围

## 核心功能

### 必须有
1. **音频上传** - 支持 MP3、WAV
2. **模板选择** - 至少 5 个默认模板
3. **视频导出** - 1080p MP4
4. **实时预览** - 编辑时即时看到效果

### 应该有
1. **自定义颜色** - 调整模板配色
2. **添加文字** - 歌名、艺人名
3. **进度条样式** - 多种进度条选择

## 非目标

### 绝对不做
- **视频剪辑功能** - 我们不是剪映
- **多轨音频混音** - 我们不是 DAW
- **AI 作曲** - 不是我们的领域
- **社交功能** - 不做社区

### 当前不做（可能未来）
- **移动端 App** - 先专注 Web
- **团队协作** - MVP 只支持个人使用
- **API 开放** - 先产品稳定再说

## 可选功能
- 多语言支持
- 深色/浅色主题切换
- 键盘快捷键

## 技术约束
- 视频导出最长 10 分钟（浏览器内存限制）
- 音频文件最大 50MB
- 免费用户 720p，付费用户 1080p
```

## AI 使用指南

当 AI 读取 SCOPE.md 时，应该：

1. **边界意识** - 遇到非目标中的功能请求，应该提醒用户
2. **优先级** - 核心功能 > 应该有 > 可选功能
3. **技术约束** - 实现时考虑列出的技术限制
4. **不要过度** - 不主动添加不在列表中的功能
