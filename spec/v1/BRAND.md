# BRAND.md 格式规范

**版本**: v1
**状态**: 稳定

## 概述

BRAND.md 定义产品的品牌调性，指导AI在写文案、UI文字、文档时保持一致的风格。

## 推荐字段

```markdown
# 品牌指南

## 语气
<!-- 描述整体沟通风格 -->

## 称呼
<!-- 用"你"还是"您"？ -->

## 关键词
<!-- 品牌常用的词汇 -->

## 禁用词汇
<!-- 永远不要使用的词 -->
```

## 可选字段

```markdown
## 视觉风格
<!-- 配色、字体偏好 -->

## 示例文案
<!-- 好的文案示例 -->

## 竞品对比
<!-- 我们的风格与竞品有何不同 -->
```

## 示例

```markdown
# Vibling 品牌指南

## 语气
友好、鼓励、不居高临下。
像一个有经验的朋友在帮你，而不是冷冰冰的工具。

## 称呼
- 用"你"而不是"您"
- 可以用"我们"表示Vibling团队

## 关键词
- 简单、快速、专业
- 创作、表达、分享
- 音乐、视觉、艺术

## 禁用词汇
- "一键XX"（太营销）
- "强大"（太虚）
- "革命性"（太夸张）
- "小白"（不尊重用户）

## 视觉风格
- 主色：深紫色 (#6B46C1)
- 背景：深色主题优先
- 字体：无衬线，现代感

## 示例文案

**好的**:
- "上传你的音乐，选个模板，3分钟搞定"
- "不需要学习复杂软件"
- "专为音乐人设计"

**不好的**:
- "一键生成专业级可视化视频！"
- "强大的AI引擎为您服务"
- "小白也能轻松上手"
```

## AI使用指南

当AI读取BRAND.md时，应该：

1. **文案生成** - 写任何面向用户的文字时遵循语气和用词
2. **代码注释** - 内部注释可以不遵循，但用户可见的字符串要遵循
3. **错误消息** - 用品牌语气写错误提示
4. **主动检查** - 写完文案后检查是否使用了禁用词汇
