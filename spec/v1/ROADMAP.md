# ROADMAP.md 格式规范

**版本**: v1
**状态**: 稳定

## 概述

ROADMAP.md 定义产品的战略方向和里程碑，帮助AI理解当前阶段和优先级。

## 必需字段

```markdown
# 产品路线图

## 当前阶段
<!-- MVP / Beta / Growth / Scale -->

## 里程碑
<!-- 至少定义当前阶段的目标 -->
```

## 推荐结构

```markdown
# 产品路线图

## 当前阶段
MVP - 验证核心价值

## 里程碑

### Phase 1: MVP (目标日期)
**目标**: 验证用户是否需要这个产品

- [x] 已完成的功能
- [ ] 进行中的功能
- [ ] 计划中的功能

### Phase 2: Beta
**目标**: 收集用户反馈，完善体验

- [ ] 功能A
- [ ] 功能B

### Phase 3: 商业化
**目标**: 验证商业模式

- [ ] 支付集成
- [ ] 定价策略

### 未来考虑
<!-- 长期想法，不承诺时间 -->
- 想法A
- 想法B
```

## 示例

```markdown
# Vibling 路线图

## 当前阶段
MVP - 验证可视化生成的核心价值

## 里程碑

### Phase 1: MVP (2025-02)
**目标**: 用户能上传音乐，选择模板，导出视频

- [x] 音频上传
- [x] 波形可视化
- [ ] 模板系统
- [ ] 视频导出

### Phase 2: Beta (2025-04)
**目标**: 提升可用性，收集反馈

- [ ] 更多模板
- [ ] 自定义颜色/字体
- [ ] 用户账号系统

### Phase 3: 商业化 (2025-06)
**目标**: 验证付费意愿

- [ ] 免费/付费模板分层
- [ ] 订阅制
- [ ] 水印移除

### 未来考虑
- AI生成模板
- 移动端支持
- 协作功能
```

## AI使用指南

当AI读取ROADMAP.md时，应该：

1. **聚焦当前阶段** - 优先完成当前Phase的任务
2. **延迟复杂度** - 如果功能在后续Phase，建议简化当前实现
3. **区分已完成/进行中** - 不要重复实现已完成的功能
4. **提醒偏离** - 如果用户请求的功能不在任何Phase中，应该询问
